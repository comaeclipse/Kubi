# SafeVision — iOS App Conversion Guide

## The Apple Guidelines Problem

Apple will **reject** a bare WKWebView wrapper of your Vercel site — guideline **4.2 Minimum Functionality** requires apps to feel native. However, SafeVision has real features (profiles, progress tracking, curated content, parental controls) that give it legitimate app value. The challenge is the delivery mechanism.

## Recommended Path: Capacitor (Ionic)

This is the lowest-friction option for the stack. Capacitor wraps the Next.js app but lets you add native plugins that satisfy Apple's requirements.

**Why it fits SafeVision:**
- UI is already mobile-responsive (Tailwind breakpoints, `use-mobile` hook, collapsible sidebar)
- shadcn/ui components are touch-friendly
- API routes stay on Vercel — the app just points at them
- Minimal code changes needed

**What you'd need to add for Apple approval:**

### 1. Native feel (guideline 4.2)
- **Push notifications** via `@capacitor/push-notifications` — notify when new videos are synced to a channel
- **Haptic feedback** on profile selection, video card taps
- **Native splash screen** and app icon
- **Offline skeleton** — show cached profile/channel list when offline instead of a blank screen

### 2. Kids category compliance (guideline 1.3)
This is critical since SafeVision is a kids' content app:
- **No third-party analytics or ads** (none currently — good)
- **No external links** that leave the app — YouTube embeds play inline, but disable any link that opens Safari
- **Privacy policy URL** required
- **Age gate** — the PIN system covers this well, but declare the age range
- **COPPA compliance** — no personal data collection from kids. Profiles store only a name and color, no email/location
- **Parental gate for purchases/settings** — the PIN gate already does this

### 3. YouTube embed concern (guideline 10.6)
Apple allows YouTube iframe embeds. However, for a better native feel and to avoid YouTube's own UI showing recommended videos (which can't be controlled for kids), consider:
- Use `youtube-ios-player-helper` via a Capacitor plugin for native YouTube playback
- Or keep iframes but set `rel=0` and `modestbranding=1`

## What changes in the codebase

### Minimal changes needed:

| Area | Change |
|------|--------|
| `next.config.ts` | Add `output: 'export'` for static export, OR keep as-is and point Capacitor at Vercel URL |
| `src/lib/video-cache.ts` | Already caches client-side — extend TTL for mobile |
| `src/components/layout/app-shell.tsx` | Add safe-area insets (`env(safe-area-inset-top)`) for notch/Dynamic Island |
| `src/app/layout.tsx` | Add `<meta name="apple-mobile-web-app-capable">`, viewport-fit=cover |
| `src/components/video/video-player.tsx` | Test fullscreen behavior in WKWebView, may need native fullscreen handling |
| New: `capacitor.config.ts` | Point to Vercel URL or local build |
| New: `ios/` directory | Generated by `npx cap add ios` |

### The two architecture options:

**Option A — Remote app (easier)**
Capacitor points at `https://safervision.vercel.app`. API routes, SSR, everything stays the same. The native shell adds push notifications, haptics, and offline handling. This is the fastest path.

**Option B — Static export (more native)**
`next build` with `output: 'export'`, bundle the static files into the app. API calls still go to Vercel. Faster initial load, works partially offline. But loses server components and would need to convert API routes to standalone or keep them on Vercel as external endpoints.

**Recommendation: Option A** — it matches the current architecture with zero API changes.

## Setup

```bash
npm install @capacitor/core @capacitor/cli
npx cap init SafeVision com.safevision.app --web-dir=out
npx cap add ios
```

Then add native plugins for push notifications, haptics, splash screen, and status bar — those are the minimum to pass Apple review as a non-trivial native app.

## Alternative: Skip the App Store entirely

If the goal is just "iOS app on the home screen," a **PWA** would work with zero code changes — add a `manifest.json` and a service worker. Kids/family members tap "Add to Home Screen" in Safari. No Apple review needed. The tradeoff is no push notifications and no App Store discoverability, but for a family-use app that may not matter.

## Bottom line

Capacitor + the existing codebase + 4-5 native plugins is the path of least resistance to a real App Store listing. The PIN gate, profile system, and curated-content model already align well with Apple's kids category requirements.
